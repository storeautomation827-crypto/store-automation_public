# Store Automation v1.01（日本語版）

店舗運営管理と営業報告自動化のための Flask ベース Web アプリケーション

## プロジェクト概要

Store Automation は、日々の店舗業務、経費追跡、ビジネスインテリジェンスを管理するための包括的なシステムです。Python と Flask で構築されており、スタッフが重要なビジネスメトリクスを Web インターフェイスから入力するだけで、バックエンド が自動的にデータ処理・検証・Excel レポート出力を行います。

このリポジトリは、自動化プラットフォームを支える「コア基盤システム」と「インフラストラクチャ」を公開しています。将来のサービス拡張や他事業への応用を見据えて設計されています。

## 主な機能

- セキュアな認証・認可 - ユーザー管理とロールベースのアクセス制御
- 複数形式対応の入力フォーム - 日次報告、経費入力、売上管理などの Web フォーム
- Excel 連携機能 - Excel ワークブックへのデータ自動追記・エクスポート
- データ検証とセキュリティ - 入力値の厳密な検証とファイルパスの安全性チェック
- 包括的なテスト - 重要なビジネスロジックを網羅するユニット・統合テスト
- 拡張性の高い設計 - モジュール化された構成で新機能追加が容易

## 始め方

### 必須環境

- Python 3.10 以上
- pip（Python パッケージマネージャ）
- Windows、macOS、Linux いずれか

### インストール手順

#### 1. リポジトリをクローン
```bash
git clone https://github.com/storeautomation827/store-automation.git
cd store-automation
```

#### 2. 仮想環境を作成・有効化
```bash
# 仮想環境を作成
python -m venv venv

# 仮想環境を有効化
# Windows の場合：
venv\Scripts\activate

# macOS / Linux の場合：
source venv/bin/activate
```

#### 3. 依存パッケージをインストール
```bash
pip install -r requirements.txt
```

#### 4. 設定ファイルを準備
```bash
# サンプル設定をコピーして実装
cp config/stores.json.example config/stores.json
cp config/users.json.example config/users.json
```

#### 5. アプリケーションを起動
```bash
python run.py
```

ブラウザで `http://localhost:5000` にアクセスすると起動します

## 設定ファイルについて

### stores.json
店舗情報を定義します。

例：
```json
{
  "stores": [
    {
      "id": "store_001",
      "name": "店舗名",
      "active": true
    }
  ]
}
```

フィールド説明：
- id: 店舗を識別する ID（重複不可）
- name: 店舗の表示名
- active: 有効・無効の切り替え

### users.json
ユーザー認証情報を管理します。

例：
```json
{
  "users": [
    {
      "username": "admin",
      "password_hash": "ハッシュ化されたパスワード",
      "role": "admin"
    }
  ]
}
```

【注意】セキュリティ事項
- パスワードは必ずハッシュ化してください（werkzeug.security を使用推奨）
- 平文パスワードを commit しないでください
- .gitignore で除外されており、GitHub に上がりません

## コア機能の詳細

### 認証モジュール（app/auth/）
- ユーザーログイン・セッション管理
- パスワード検証
- ユーザー情報の永続化

### Excel 操作モジュール（app/excel/）

Scanner（スキャナー）
- Excel ファイルを読み込んでデータを解析
- 複数シートに対応

Writer（ライター）
- データを Excel ワークブックに書き込み
- セルレベルの細かい制御が可能

Backup（バックアップ）
- 重要な Excel ファイルを自動バックアップ
- トラブル時の復旧に対応

Path Guard（パス保護）
- ファイルパスの検証
- ディレクトリ走査攻撃（Path Traversal）を防止

### サービスモジュール（app/services/）
- Database: SQLite ORM とスキーマ初期化
- データ集計とビジネスロジック

### Web ルーティング（app/web/）
- フォーム送信のエンドポイント
- データ取得・レポート出力

## 重要な注意事項

このコードは「デモンストレーション・ポートフォリオ用途」です。実運用データ、店舗固有の情報、プライバシー情報は抽象化・削除されています。

- 本番 Web サーバ - Gunicorn、uWSGI などで実装
- レート制限 - API 使用制限の実装
- HTTPS・CORS - セキュアな通信設定
- スケーリング - 複数インスタンス対応
