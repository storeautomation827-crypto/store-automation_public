# Store Automation v1.01（日本語版）

飲食店の店舗業務を効率化するための Flask ベース Web アプリケーション

## プロジェクト概要

Store Automation は、飲食店における日々の売上管理、経費入力、営業報告などの業務を効率化することを目的に開発している Web アプリケーションです。

アルバイト先の店舗では、日々の売上や営業報告、経費情報などを Excel 中心で管理しており、入力作業や確認作業に手間がかかるという課題がありました。そこで、既存の Excel 運用を大きく変えずに、Web 画面から必要な情報を入力・保存・確認し、Excel ファイルへ反映できる仕組みを構築しています。

本リポジトリでは、ポートフォリオ用として Flask アプリケーションの基幹部分、入力・保存・確認フロー、Excel 連携、バックアップ処理、テストコードを中心に掲載しています。

## 前作との関係

以前は、LINE Bot と OCR を用いて、画像から売上情報を読み取り、Excel へ反映する仕組みを作成しました。

関連リポジトリ：  
[store-automation-ocr](https://github.com/storeautomation827-crypto/store-automation-ocr)

この前作では、画像から情報を抽出して Excel 入力を支援することを目的としていました。一方で、LINE Bot 中心の操作では、入力項目や確認項目が増えた場合に、メッセージのやり取りが多くなり、操作の手間が増えやすいという課題がありました。

そこで現在は、より多くの入力項目やメニューを扱いやすくし、機能追加やアップデートを行いやすい形にするため、Flask ベースの Web アプリケーションとして作り直しています。

## 開発の目的

このアプリケーションでは、次のような課題の解決を目指しています。

- 手作業による Excel 入力の負担を減らす
- 売上・経費・営業報告などの情報を一元的に管理する
- スマートフォンや PC のブラウザから入力できるようにする
- 入力項目やメニューを増やしやすい画面構成にする
- 既存の Excel 管理方法を活かしながら、業務改善を行う

## 主な機能

- ログイン機能
- 店舗情報の管理
- 日報・経費・営業報告などの入力フォーム
- 入力内容の SQLite 保存
- 入力履歴の確認
- Excel ファイルへのデータ反映
- Excel 書き込み前のバックアップ作成
- 入力値や書き込み対象ファイルの確認
- テストコードによる主要処理の検証

## 処理の流れ

1. スタッフがログイン
2. 入力種別を選択
3. 日報・経費・営業報告などの情報を入力
4. 入力内容を SQLite に保存
5. 必要に応じて入力履歴を確認
6. Excel ファイルへ反映
7. 反映前にバックアップを作成

ブラウザから入力された情報は、Flask アプリケーション側で受け取り、サーバー側の SQLite データベースに保存する構成にしています。

## 工夫した点

### 1. LINE Bot から Web アプリへの発展

前作では LINE Bot を使って入力や確認を行っていましたが、項目が増えるほどメッセージの往復が多くなり、操作が複雑になりやすいという課題がありました。

本アプリでは、ブラウザ上で入力フォームやメニューを表示できるため、日報・経費・営業報告など複数の機能をまとめて扱いやすくしています。

### 2. 既存の Excel 運用を活かした設計

実際の店舗業務では、すでに Excel を使った管理フローが存在していました。そのため、既存の運用を完全に置き換えるのではなく、Web 入力と Excel 反映を組み合わせることで、現場に導入しやすい構成を意識しました。

### 3. 書き込み前バックアップによるデータ保護

Excel ファイルへデータを反映する前にバックアップを作成し、誤入力や書き込み失敗が発生した場合でも元のファイルを保護できるようにしています。

### 4. 公開範囲を分けた構成

このリポジトリでは、実店舗名、実データ、実際に使用している Excel ファイル、店舗固有のセル位置情報などは公開していません。ポートフォリオとして見せる部分と、実運用に関わる情報を分けて管理しています。

### 5. 機能ごとに分離した構成

認証、Excel 操作、データベース処理、Web ルーティングなどを分けて実装し、後から機能追加や修正がしやすい構成を意識しました。

### 6. テストによる安全性確認

Excel 書き込み処理、データベース操作、入力フォーム処理など、重要な処理についてはテストコードを作成し、変更後も動作を確認できるようにしています。

## プロジェクト構成

主なファイル・フォルダは以下の通りです。

- `app/`
  - Flask アプリケーションの本体です。
  - 認証、Excel 操作、データベース処理、画面ルーティングをここに分けて実装しています。

- `app/auth/`
  - ログイン処理、ユーザー管理、セッション管理に関する処理を置いています。

- `app/excel/`
  - Excel ファイルの読み込み、書き込み、バックアップ処理を置いています。

- `app/services/`
  - SQLite への保存処理や、入力データを扱う処理を置いています。

- `app/web/`
  - Flask のルート処理や、フォーム送信の受け取り処理を置いています。

- `templates/`
  - 画面表示に使用する HTML ファイルを置いています。

- `static/`
  - CSS や JavaScript など、画面側で使うファイルを置いています。

- `tests/`
  - データベース操作、Excel 書き込み、フォーム処理などを確認するテストコードを置いています。

- `scripts/`
  - 開発や動作確認に使う補助スクリプトを置いています。

- `config/`
  - 店舗情報やユーザー情報などの設定ファイルを置いています。
  - 実際に使用する設定ファイルは、公開用ファイルとは分けて管理します。

- `requirements.txt`
  - アプリケーションの実行に必要な Python パッケージをまとめています。

- `run.py`
  - Flask アプリケーションを起動するためのファイルです。

## 始め方

### 必須環境

- Python 3.10 以上
- pip
- Windows / macOS / Linux

### 1. リポジトリをクローン

```bash
git clone https://github.com/storeautomation827-crypto/store-automation_public.git
```

```bash
cd store-automation_public
```

### 2. 仮想環境を作成・有効化

```bash
python -m venv venv
```

Windows の場合：

```bash
venv\Scripts\activate
```

macOS / Linux の場合：

```bash
source venv/bin/activate
```

### 3. 依存パッケージをインストール

```bash
pip install -r requirements.txt
```

### 4. 設定ファイルを準備

必要に応じて、サンプル設定ファイルをコピーして使用します。

```bash
cp config/stores.json.example config/stores.json
```

```bash
cp config/users.json.example config/users.json
```

Windows のコマンドプロンプトで実行する場合は、以下のようにコピーできます。

```cmd
copy config\stores.json.example config\stores.json
```

```cmd
copy config\users.json.example config\users.json
```

### 5. アプリケーションを起動

```bash
python run.py
```

起動後、ブラウザで以下にアクセスします。

```text
http://localhost:5000
```

## 設定ファイルについて

### stores.json

店舗情報を定義します。

```json
{
  "stores": [
    {
      "id": "store_001",
      "name": "Sample Store",
      "active": true
    }
  ]
}
```

主な項目は以下の通りです。

- `id`
  - 店舗を識別する ID です。

- `name`
  - 店舗の表示名です。

- `active`
  - 店舗を有効にするかどうかを管理します。

### users.json

ユーザー認証情報を管理します。

```json
{
  "users": [
    {
      "username": "admin",
      "password_hash": "hashed_password",
      "role": "admin"
    }
  ]
}
```

主な項目は以下の通りです。

- `username`
  - ログインに使用するユーザー名です。

- `password_hash`
  - ハッシュ化されたパスワードです。

- `role`
  - ユーザーの権限を管理します。

### セキュリティ上の注意

- パスワードは必ずハッシュ化してください
- 平文パスワードを commit しないでください
- 実運用の設定ファイルやデータは GitHub に公開しないでください
- `.gitignore` により、個別設定ファイルは除外しています
